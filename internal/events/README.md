# Events Repository - SQLc Implementation

This package implements the `datastore.EventRepository` interface using sqlc-generated queries.

## Structure

```
internal/events/
├── impl.go                    # Service implementation (18 methods)
├── helpers.go                 # Type conversion utilities
├── queries.sql               # SQL query definitions (19 queries)
├── migration.md              # Migration progress tracker
├── impl_test.go              # Comprehensive test suite (to be created)
└── repo/                     # Generated by sqlc (gitignored)
    ├── db.go
    ├── models.go
    ├── querier.go
    └── queries.sql.go
```

## Usage

```go
import (
    "github.com/frain-dev/convoy/internal/events"
    "github.com/frain-dev/convoy/pkg/log"
    "github.com/jackc/pgx/v5/pgxpool"
)

// Initialize the service
logger := log.NewLogger(os.Stdout)
db := pgxpool.Connect(ctx, connString)
eventRepo := events.New(logger, db)

// Use the repository
event, err := eventRepo.FindEventByID(ctx, projectID, eventID)
```

## Key Features

- **SQLc-based**: Type-safe SQL queries with compile-time validation
- **Dual query paths**: Optimized pagination with EXISTS vs CTE+JOIN
- **Batch processing**: 30K partition size for event_endpoints
- **Transaction support**: Preserves nested transaction context
- **19 queries**: Consolidated from 14+ variants in legacy implementation

## Migration from Legacy

Migrating from `database/postgres/event.go` to this implementation:

**Before:**
```go
import "github.com/frain-dev/convoy/database/postgres"
eventRepo := postgres.NewEventRepo(db)
```

**After:**
```go
import "github.com/frain-dev/convoy/internal/events"
eventRepo := events.New(logger, db)
```

## Performance Considerations

1. **EXISTS vs CTE path**: Empty search queries use EXISTS subquery to leverage indexes
2. **Batch processing**: Event endpoints inserted in 30K partitions to avoid memory issues
3. **Cursor pagination**: Efficient navigation using event IDs as cursors
4. **CASE expressions**: Conditional filters compiled into single query instead of dynamic SQL building
