# Event Deliveries: PostgreSQL to sqlc Migration

## Status: Complete (including tests)

## What was migrated
- `database/postgres/event_delivery.go` (1520 lines, sqlx) → `internal/event_deliveries/` (sqlc + pgx)
- `database/postgres/event_delivery_test.go` (integration test with build tag) → deleted

## New files
- `internal/event_deliveries/queries.sql` - 19 sqlc-annotated SQL queries
- `internal/event_deliveries/helpers.go` - Row conversion, JSONB helpers, interval padding
- `internal/event_deliveries/impl.go` - Service implementing `datastore.EventDeliveryRepository`
- `internal/event_deliveries/impl_test.go` - 18 test functions, 39 subtests
- `internal/event_deliveries/repo/` - Auto-generated by sqlc (db.go, models.go, querier.go, queries.sql.go)

## Key changes
- Replaced sqlx with sqlc-generated type-safe queries
- Uses pgx/v5 directly instead of database/sql
- Batch inserts use `pgx.Batch` instead of sqlx `NamedExec`
- Pagination uses single CASE-based query instead of dual forward/backward string replacement
- Intervals use 4 separate queries (daily/weekly/monthly/yearly) instead of fmt.Sprintf
- Service holds `*hooks.Hook` for `EventDeliveryUpdated` fire
- Constructor: `event_deliveries.New(logger, db)` instead of `postgres.NewEventDeliveryRepo(db)`
